## Ornith 9B 與 Qwen 9B 效能評測簡報：本地 8GB GPU LLM 選擇指南

本簡報旨在深入分析 Ornith 9B 與 Qwen 9B（3.5 版本）在本地 8GB VRAM GPU 環境下的效能表現。這兩款模型均採用 GGUF 格式與 4-bit 量化（Q4KM），目標對象為在硬體資源有限的情況下尋求高效能 AI 方案的使用者。

### 執行摘要

根據詳細的基準測試與實際編碼挑戰，Ornith 9B 在整體穩定性、工具調用準確性以及 Python 編碼邏輯方面優於 Qwen 9B。儘管兩者在基礎生成速度上表現相當，且在處理超大文本的「大海撈針」測試中均表現不佳，但 Ornith 9B 在實際應用場景（如代理任務與功能性編碼）中展現出更高的可靠性。Qwen 9B 雖然在 UI/UX 設計美感上略勝一籌，但常因語法錯誤或工具調用格式問題導致任務失敗。

### 關鍵主題深入分析

#### 1. 效能與推論速度

在 8GB VRAM 的環境下，兩款模型的推論效能幾乎平分秋色：
*   **預填速度 (Prefill)：** 隨著輸入內容增加，速度會逐漸下降，但在 32K 標記（tokens）後趨於平穩。
*   **解碼速度 (Decode)：** 兩者均維持在每秒約 36 至 37 個標記。Ornith 在此項測試中僅以 1 到 2 個標記的微弱優勢領先。

#### 2. 記憶力與長文本處理（大海撈針測試）

這兩款 9B 級別的模型在處理極大上下文（如 256K、128K 甚至 64K 標記）時，擷取特定數據的能力均顯著不足。
*   **測試結果：** 模型在文本的開頭與結尾表現較好，但在中間部分經常迷失。
*   **輸出的品質：** 即使找到了數據，輸出內容也往往不夠簡潔，常混雜無關的文本背景（如 Linux 內核文件內容）。
*   **結論：** 對於需要精準檢索海量資料的任務，9B 級別的模型目前仍不具備足夠的可靠性。

#### 3. 代理能力與工具調用 (Agency Benchmark)

此測試將模型置於虛擬公司沙盒中，測試其使用工具、鏈接任務及處理邏輯的能力。

| 模型 | 成功率 | 表現分析 |
| :--- | :--- | :--- |
| **Ornith 9B** | **91%** (20/22) | 表現極佳，能有效鏈接多個工具完成任務（如貨幣轉換）。 |
| **Qwen 9B** | 較低 | 邏輯雖理解任務，但常因**語法格式錯誤**（格式洩漏）導致無法正確觸發工具。 |

#### 4. 編碼能力測試

測試涵蓋了 OpenAI Human Eval（164 項 Python 挑戰）及兩項實際開發任務。

*   **Python 挑戰 (Human Eval)：**
    *   Ornith 9B 通過率為 91%（通過 148 題）。
    *   Qwen 9B 通過率為 87%（通過 143 題）。
*   **實際開發測試 A：開支追蹤器 (Expense Tracker)**
    *   Ornith 9B 迅速完成功能性代碼，僅需少量提示即可修正 Bug。
    *   Qwen 9B 雖然 UI 設計較佳，但核心功能（如類別下拉選單與數據提交）存在持久性 Bug，難以透過提示修復。
*   **實際開發測試 B：記憶比對遊戲 (Memory Match Game)**
    *   Ornith 9B 初稿功能完整但視覺簡單，需額外提示修復隱藏卡片的邏輯。
    *   Qwen 9B 在視覺動畫與樣式設計上明顯優越，雖初期有完成狀態的邏輯錯誤，但修復速度快。

### 重要語錄與背景分析

> **「即使在文本開頭，它也只在三次嘗試中成功找到了一次……當它找不到時，就會迷失在文件的背景脈絡中。」**

*   **背景：** 這是針對 Ornith 在「大海撈針」測試中的評語。
*   **分析：** 這凸顯了小型模型（9B）在處理複雜長文本時的局限性，不僅是找不到資訊，還會產生幻覺或受背景雜訊干擾。

> **「Qwen 並非敗在智力不足，而是敗在語法崩潰，當它試圖調用工具時，格式發生了洩漏。」**

*   **背景：** 針對代理能力測試的觀察。
*   **分析：** 這說明了 Qwen 在指令遵循（Instruction Following）的精確度上仍有提升空間，雖然它理解該做什麼，卻無法以正確的機器可讀格式輸出。

> **「Ornith 到達結果的速度比 Qwen 快得多，這在這次測試中是明顯的勝出。」**

*   **背景：** 針對開支追蹤器編碼任務的總結。
*   **分析：** 在實際生產力環境中，邏輯的正確性與修復 Bug 的效率比初步的 UI 視覺效果更為重要。

### 洞察與行動建議

#### 1. 根據使用場景選擇模型

*   **優先選擇 Ornith 9B 的場景：**
    *   需要高度可靠的工具調用或自動化工作流（Agentic Workflows）。
    *   專注於 Python 代碼開發與邏輯實現。
    *   追求開發速度，減少反覆修錯的成本。
*   **優先選擇 Qwen 9B 的場景：**
    *   注重 Web 前端設計感、UI 佈局與動畫效果。
    *   一般性的創意寫作或對格式精確度要求較低的對話。

#### 2. 硬體與環境配置建議

*   **GPU 顯存：** 8GB VRAM 使用者應堅持選用 4-bit 量化版本，以平衡效能與記憶體佔用。
*   **系統記憶體：** 建議配備 32GB 系統記憶體，以便在 VRAM 不足時提供緩衝空間。
*   **上下文管理：** 鑑於 9B 模型在長文本檢索上的弱點，應避免在單次 Prompt 中置入過量背景資料（建議保持在 32K 以內以獲得最佳效果）。

#### 3. 未來測試方向

若追求更強的長文本檢索能力，應考慮轉向 27B（如 Gemma）或 35B 以上級別的稠密模型（Dense Models），小型模型在目前的技術限制下，尚無法完美處理極大規模的上下文檢索。
