    *   A: The program is suitable for beginners with basic application programming skills. Advanced mathematics, traditional AI, or machine learning background is not required. The beauty of AI allows leveraging it without deep theoretical ML knowledge.

    *   A: Agent evaluation is covered within ADK's local development UI (e.g., the evaluation tab). Agent API design patterns may be touched upon in workshops, but more in-depth material will be available in the optional Google Skills resources. Full coverage of all design patterns may not be possible in instructor-led sessions due to time constraints, but information will be accessible.

*   **Q: What are the prerequisites to learn ADK?**
    *   A: The main prerequisite is the ability to program. Basic knowledge of Python is a great beginning, as some courses will be Python-focused, though ADK supports other languages.

*   **Q: Which database is preferred for agentic AI?**
    *   A: All Google Cloud databases are AI-ready (vector/embeddings ready). The program will focus on Postgress-compliant databases like AlloyDB and Cloud SQL for ease of use, as they are popular among developers.

*   **Q: How much does Cloud Run cost to run an MCP server or agents?**
    *   A: Cloud Run has a very healthy free tier, allowing thousands of calls per day for free. This should cover development and testing without incurring costs. Charges would only apply if traffic exceeds this free tier, which is expected for production applications.

*   **Q: Can AI orchestration be used in areas of work like cyber security and high-end sales?**
    *   A: Absolutely yes. In cyber security, agents can identify/triage incidents, and monitor vulnerabilities (either DIY or using existing Google Cloud agents). In high-end sales, agents can assist with sales/marketing plans using real-time data. The approach should be goal-based, applying domain knowledge.

*   **Q: How to decide on a model to use, for example, for an accounts-related feature, and run data tests for the best LLM?**
    *   A: It's recommended to start with a model like Gemini 3 Flash, which offers a great price-to-performance ratio. The initial focus should be on building the agentic system correctly. Once a baseline is established, use evaluation tools (like ADK's evaluation tab) with "eval sets" to compare models iteratively and see which performs best for specific use cases.

*   **Q: Can we get exposed to RLHF-based tuning of agents? How effective and user-friendly is it?**
    *   A: Google Cloud's Vertex AI offers RLHF services for model tuning. While not part of the core Cohort 1 curriculum, it is available as optional material within Google Skills labs. It's important to be aware that RLHF and fine-tuning may incur additional costs.

*   **Q: If we run agents locally, is it keeping data to itself or will it be tracked back to Google if we use its cellular?**
    *   A: It depends on the specific AI model used. Participants are encouraged to always review the terms and conditions and clauses of each model (first-party or third-party). Enterprise models generally guarantee that user data will not be used. However, preview or free models might have different clauses.

*   **Q: You mentioned earlier that we cannot use agents in regulated environments. Do you mean life sciences and healthcare domains?**
    *   A: There isn't a hard "yes" or "no" for regulated environments, but rather a need to understand specific regulations. This can include:
        *   **Data Residency:** Ensuring all system components (databases, AI models) are deployed within specific regional/country boundaries.
        *   **Human-in-the-loop:** For sensitive recommendations (e.g., medical), a human review step might be legally required.
        *   **Explainable AI:** If regulations demand explanation for decisions, agents might need to be designed as workflow-based systems (e.g., using ADK's workflow capabilities) rather than fully autonomous "opaque boxes" to allow for more clear tracing of execution. Life sciences and healthcare are examples of domains that often have such regulations.
